## v1.6.0

Minor release for **GitHub Pulls Show Reviewers** focused on reducing cold
pull-list request volume while keeping reviewer visibility as the only product
scope. Requested reviewers, requested teams, and completed review state remain
the only surfaced PR-list metadata.

### Highlights

- Cold PR-list rendering now fetches page-level REST pull metadata once per
  page/account, then reuses requested reviewer, requested team, and author
  metadata for matching row summaries.
- Rows covered by the page-level metadata skip the per-row pull endpoint and
  only fetch review history, reducing typical first-page cold requests from
  `2N` to `1 + N`.
- Rows outside the first REST pull-list page still fall back to the original
  per-row `pull + reviews` path, preserving support for filtered, searched, and
  older list pages.

### Authentication

- No permission, token-scope, storage-schema, or host-permission changes.
- Public repositories still work without signing in when GitHub's
  unauthenticated REST API allows it.
- Private repositories continue to route through the maintainer-owned GitHub
  App account flow, and page-level metadata uses the same background-worker
  token handling and refresh behavior as row summaries.

### Quality & Packaging

- Added unit coverage for the REST metadata batch, runtime message contract,
  and metadata reuse in reviewer row summaries.
- Added packaged-extension fixture coverage for the new batch request path.
- Reworked Chrome Web Store screenshot capture around deterministic fixture
  pages and refreshed the submission screenshots to avoid live dummy PRs.
- Chrome Web Store submission materials now point at the v1.6.0 package name.

### Scope Notes

- Reviewer visibility on GitHub pull request list pages remains the only
  product goal.
- Checks, mergeability, labels, assignees, and general PR dashboard features
  remain out of scope.
