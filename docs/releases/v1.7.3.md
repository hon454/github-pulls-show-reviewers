## v1.7.3

Patch release for **GitHub Pulls Show Reviewers** focused on reviewer row
stability, keyboard focus visibility, typed diagnostics, and release
operations. The reviewer feature scope stays unchanged: requested reviewers,
requested teams, and completed review state remain the only surfaced PR-list
metadata.

### Highlights

- Reviewer rows now ignore GitHub `relative-time` mutations when deciding
  whether to refetch reviewer data, reducing unnecessary repeat fetches while
  the pull request list updates in place.
- Injected reviewer UI and access guidance now include explicit
  `focus-visible` styles so keyboard navigation remains visible on GitHub pull
  request list pages.
- User-facing GitHub schema parse failures now map to typed diagnostics,
  making repository and authentication troubleshooting more consistent.

### Authentication

- No permission, token-scope, storage-schema, or host-permission changes.
- Public repositories still work without signing in when GitHub's
  unauthenticated REST API allows it.
- Private repositories continue to use the maintainer-owned GitHub App account
  flow with `Pull requests: Read` access only.

### Quality & Packaging

- Added a weekly `pnpm audit` workflow for dependency visibility without
  changing runtime behavior.
- Tightened the manual Chrome Web Store upload checklist with version
  alignment checks across `package.json`, the zip filename, the Git tag, and
  release notes.
- Chrome Web Store submission materials now reference the v1.7.3 package
  name.

### Scope Notes

- Reviewer visibility on GitHub pull request list pages remains the only
  product goal.
- Checks, mergeability, labels, assignees, and general PR dashboard features
  remain out of scope.
