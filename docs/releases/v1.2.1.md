## v1.2.1

Patch release for **GitHub Pulls Show Reviewers** focused on tighter inline row
presentation, clearer private-organization setup copy, and safer Playwright
execution guidance for maintainers.

### Highlights

- PR list rows now hide empty inline reviewer sections instead of rendering
  `No requested reviewers` and `No completed reviews`, which keeps the metadata
  on one line more often while preserving the `Requested:` and `Reviewed:`
  labels when data exists.
- The options page and README now call out that organization-owned private
  repositories may require an organization owner to install the GitHub App
  first.

### Developer Experience

- End-to-end test commands are split into `pnpm test:e2e:build`,
  `pnpm test:e2e:run`, and `pnpm test:e2e:grep "<pattern>"` so filtered
  Playwright runs no longer need ad hoc command rewriting.
- The Chrome Web Store notes and README now document the supported filtered
  Playwright workflow explicitly.

### Quality & Packaging

- Added regression coverage for rows with no requested reviewers and no
  completed reviews.
- Refreshed the options-page Chrome Web Store screenshot so store assets match
  the current private-organization installation guidance.

### Scope Notes

- No change to review-state mapping, chip colors, or reviewer link behavior.
- No Marketplace packaging changes; Chrome Web Store packaging remains the
  distribution target.
