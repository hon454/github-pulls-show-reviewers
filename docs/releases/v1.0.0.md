## v1.0.0

Initial public release of **GitHub Pulls Show Reviewers**, a Chrome extension that makes reviewer visibility obvious on GitHub pull request list pages.

This release focuses on one narrow workflow: showing requested reviewers, requested teams, and each reviewer's latest completed review state directly in the PR list, without turning the page into a general PR dashboard.

### Highlights

- Show requested user reviewers directly in pull request list rows
- Show requested team reviewers directly in pull request list rows
- Show each reviewer's latest completed review state:
  - Approved
  - Changes requested
  - Commented
  - Dismissed
- Reuse per-page cache entries to avoid duplicate row-level fetches
- Re-run safely across GitHub SPA navigation and DOM mutations

### Authentication

- Public repositories work on the no-token path when GitHub's unauthenticated REST access is available
- Private repositories are supported through a fine-grained personal access token with minimal read access
- The options page validates saved tokens and can run repository-aware diagnostics against the same API paths used by the content script

### Quality & Packaging

- Added fixture-backed regression coverage for GitHub DOM variations
- Added Playwright extension coverage for rendering and boot behavior
- Added GitHub Actions release packaging for the Chrome extension zip
- Added Chrome Web Store assets, icons, and submission/privacy documentation

### Scope Notes

This extension intentionally stays narrow in scope for the first release.

- In scope: requested reviewers, requested teams, and completed review state
- Out of scope: checks, mergeability, labels, assignees, and dashboard-style PR metadata
