## v1.7.1

Patch release for **GitHub Pulls Show Reviewers** focused on stale reviewer
request recovery and refreshed extension icon assets. The reviewer feature
scope stays unchanged: requested reviewers, requested teams, and completed
review state remain the only surfaced PR-list metadata.

### Highlights

- Reviewer fetches now recover when GitHub returns stale requested-reviewer
data by falling back to pull request review requests for the affected row.
- Requested reviewer and team visibility is more reliable after reviewers are
removed, re-requested, or changed around GitHub's pull list cache behavior.
- Extension icon assets have been refreshed from the source SVG, keeping the
packaged Chrome icon set aligned across all supported sizes.

### Authentication

- No permission, token-scope, storage-schema, or host-permission changes.
- Public repositories still work without signing in when GitHub's
unauthenticated REST API allows it.
- Private repositories continue to use the maintainer-owned GitHub App account
flow with `Pull requests: Read` access only.

### Quality & Packaging

- Added regression coverage for stale requested-reviewer API responses and
review-request fallback behavior.
- Updated manual testing notes for stale review-request recovery.
- Chrome Web Store submission materials now point at the v1.7.1 package name.

### Scope Notes

- Reviewer visibility on GitHub pull request list pages remains the only
  product goal.
- Checks, mergeability, labels, assignees, and general PR dashboard features
remain out of scope.
