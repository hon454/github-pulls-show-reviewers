## v1.4.5

Patch release for **GitHub Pulls Show Reviewers** focused on auth safety and
release-readiness documentation cleanup. The reviewer feature scope stays
unchanged: requested reviewers, requested teams, and completed review state
remain the only surfaced PR-list metadata.

### Highlights

- Successful GitHub refresh-token responses that omit rotation fields now keep
  the account's existing refresh token and refresh-token expiry instead of
  clearing them from local storage.
- Chrome Web Store storage-permission copy now lists all persisted reviewer
  display preferences, including `openPullsOnly`.
- The privacy policy contact section now points users to the public GitHub
  Issues tracker instead of a publication-time placeholder.

### Authentication

- No permission, token-scope, storage-schema, or host-permission changes.
- Existing private-repository accounts no longer lose their stored refresh
  token when GitHub returns a successful non-rotating refresh response.
- Public repositories still work without signing in when GitHub's
  unauthenticated REST API allows it.

### Quality & Packaging

- Added regression coverage for rotated and non-rotated refresh-token success
  responses.
- Updated release submission materials so the Chrome Web Store packet reflects
  the shipped storage behavior and support channel.

### Scope Notes

- Reviewer visibility on GitHub pull request list pages remains the only
  product goal.
- Checks, mergeability, labels, assignees, and general PR dashboard features
  remain out of scope.
