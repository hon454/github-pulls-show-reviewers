## v1.3.0

Minor release for **GitHub Pulls Show Reviewers** focused on a denser default reviewer row plus lightweight display controls for teams that want more or less inline detail.

### Highlights

- Pull request list rows now render a single `Reviewers:` section instead of separate requested/reviewed groups.
- User reviewers now render as avatar-first chips, with the border and optional badge carrying the current attention and review-state signals.
- The options page now includes display preferences so users can show reviewer names and hide or show review-state badges.

### Authentication

- Public repositories still work without signing in.
- Private repositories still use the maintainer-owned GitHub App via OAuth Device Flow with `Pull requests: Read` only.
- Display preferences are stored separately from connected accounts under the local `preferences` key.

### Quality & Packaging

- Added regression coverage for preference storage, live rerendering on preference changes, and the avatar/name-pill DOM variants.
- Refreshed Chrome Web Store notes, submission copy, privacy disclosure, and manual Chrome testing guidance so release materials match the shipped display settings UX.
- Refreshed store screenshots so the listing shows the merged `Reviewers:` row, the optional name-pill layout, and the updated options page.

### Scope Notes

- Reviewer visibility remains the only product goal.
- Requested reviewers, requested teams, and completed review state remain the only pull-request metadata surfaced inline.
- Checks, mergeability, labels, assignees, and general PR dashboard features remain out of scope.
