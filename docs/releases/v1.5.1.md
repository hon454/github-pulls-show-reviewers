## v1.5.1

Patch release for **GitHub Pulls Show Reviewers** focused on clearer options
page failure handling. The reviewer feature scope stays unchanged: requested
reviewers, requested teams, and completed review state remain the only surfaced
PR-list metadata.

### Highlights

- Options-page async actions now surface failures inline instead of silently
  resetting controls after failed save, sign-out, or diagnostics actions.
- Account, diagnostics, and display-settings controls now keep their busy and
  error states explicit so users can tell when an operation did not complete.

### Authentication

- No permission, token-scope, storage-schema, or host-permission changes.
- GitHub App account storage and token-refresh behavior are unchanged.
- Public repositories still work without signing in when GitHub's
  unauthenticated REST API allows it.

### Quality & Packaging

- Added options-page regression coverage for async action failures across
  account, diagnostics, and display-settings workflows.
- Chrome Web Store submission materials now point at the v1.5.1 package name.

### Scope Notes

- Reviewer visibility on GitHub pull request list pages remains the only
  product goal.
- Checks, mergeability, labels, assignees, and general PR dashboard features
  remain out of scope.
