## v1.5.0

Minor release for **GitHub Pulls Show Reviewers** focused on private-repository
access guidance reliability. The reviewer feature scope stays unchanged:
requested reviewers, requested teams, and completed review state remain the
only surfaced PR-list metadata.

### Highlights

- Private-repository access can now self-heal stale GitHub App installation
  coverage before showing a repository as uncovered. When an account owns a
  selected installation for the same owner, the extension refreshes installation
  and selected-repository metadata in the background, then re-resolves reviewer
  access for the page.
- Rate-limit banners now include GitHub response details when available,
  including the used/limit count and a relative reset time, while keeping the
  previous static fallback copy for partial or missing headers.
- Access-banner failure flows are now covered in the packaged MV3 extension for
  signed-out rate limits and signed-in uncovered-repository guidance.

### Authentication

- No permission, token-scope, storage-schema, or host-permission changes.
- Installation refreshes run through the background service worker so GitHub App
  access tokens still do not enter the content-script context.
- Rate-limit snapshots are carried only in the in-memory fetch failure envelope
  and banner state; nothing new is persisted.
- Public repositories still work without signing in when GitHub's
  unauthenticated REST API allows it.

### Quality & Packaging

- Added unit coverage for installation refresh, account re-resolution,
  rate-limit snapshot serialization, banner aggregation, and rate-limit copy.
- Added packaged-extension Playwright coverage for signed-out 429 and signed-in
  404 access-banner flows.
- Chrome Web Store submission materials now point at the v1.5.0 package name.

### Scope Notes

- Reviewer visibility on GitHub pull request list pages remains the only
  product goal.
- Checks, mergeability, labels, assignees, and general PR dashboard features
  remain out of scope.
