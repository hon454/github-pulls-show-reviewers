## v1.11.0

Minor release for **GitHub Pulls Show Reviewers** focused on making reviewer
loading request volume more explicit and bounded. This release keeps the
product scope unchanged: requested reviewers, requested teams, and completed
review state remain the only pull-list metadata surfaced by the extension.

### Highlights

- Ambiguous requested-and-reviewed users now read at most two GitHub issue-event
  pages when deciding whether a later `review_requested` event should keep the
  requested reviewer refresh badge.
- If a confirming review-request event is beyond that bounded lookup, the row
  falls back to the completed review state instead of showing an uncertain
  re-requested marker.
- Reviewer metadata batching now publishes the fixed REST page budgets used by
  reviewer loading, making the shared pull-list discovery limit explicit in code
  and tests.

### Authentication

- No permission, token-scope, storage-schema, or host-permission changes.
- Public repositories still work without signing in when GitHub's
  unauthenticated REST API allows it.
- Private repositories continue to use the maintainer-owned GitHub App account
  flow with `Pull requests: Read` access only.
- Reviewer fetching and fallback-account behavior are unchanged.

### Quality & Packaging

- Added regression coverage for the two-page issue-event lookup bound and the
  completed-review fallback when the confirming event is outside that bound.
- Added coverage to ensure issue-event pagination does not follow non-GitHub
  API links with extension auth headers.
- Documented the cold-row request budget and older visible-row fallback behavior
  in the implementation notes.
- Release packaging behavior is unchanged: GitHub Actions still runs the
  release preflight, `pnpm verify:release`, and Chrome zip creation before
  attaching the package to the GitHub Release.

### Scope Notes

- Reviewer visibility on GitHub pull request list pages remains the only
  product goal.
- Checks, mergeability, labels, assignees, and general PR dashboard features
  remain out of scope.
