## v1.0.1

Patch release for **GitHub Pulls Show Reviewers** focused on safer private-repository setup and a tighter release process, while keeping the product scope centered on reviewer visibility inside GitHub pull request lists.

### Highlights

- Added scoped personal access token settings for `owner/*` and `owner/repo` matching
- Improved the options flow so repository diagnostics can resolve the best matching stored token for the active repository
- Added a direct shortcut to GitHub's fine-grained PAT creation flow from the options page
- Kept reviewer rendering scope unchanged: requested reviewers, requested teams, and completed review state remain the only PR-list metadata shown

### Authentication

- Public repositories still support the no-token path when GitHub's unauthenticated REST access is available
- Private repositories now have a clearer repository-aware token setup flow with narrower token selection semantics
- Matched scoped tokens are used deterministically for repository diagnostics and reviewer fetches

### Quality & Packaging

- Added regression coverage for scoped token storage helpers and repository-aware options behavior
- Added structured tag-based release notes support under `docs/releases/`
- Added release verification workflow steps that run lint, typecheck, unit tests, and Playwright coverage before packaging the Chrome zip

### Scope Notes

This extension still intentionally stays narrow in scope.

- In scope: requested reviewers, requested teams, and completed review state
- Out of scope: checks, mergeability, labels, assignees, and dashboard-style PR metadata
